2. Connexion



Pour accéder à la page de gestion d'Octopus il vous faut entrer le lien suivant : http://[IP]]/ADMIN/ .

Afin de pouvoir entrer dans l'interface de gestion d'Octopus une authentification est demandée. Elle est du type LDAP et est connectée directement à l'active directory.

Une restriction supplémentaire a été ajoutée. Seuls les utilisateurs se trouvant dans une OU spécifique peuvent accéder à celui-ci.

Une fois la connexion réussie vous serez redirigé vers la page d'accueil.


Info

Un Utilisateur local est également disponible dans le cas où LDAP ne fonctionnerait plus

3.4. OctoPage



3.4.1. Ajouter une page

Pour ajouter une nouvelle page OctoPage cliquez sur l'icône et un pop-up apparaîtra.

Pour la création d'une page, il vous est demandé plusieurs choses :

  • Nom de la page : Définit le nom de la page web. C'est celui-ci que vous devrez entrer dans l'URL pour pouvoir y accéder.
  • Temps entre les pages : Définit le temps entre chaque changement de page. Celui-ci sera appliqué à l'ensemble des dashboards.


Info

Le temps entre les pages est par défaut de 30 secondes. Si cela vous convient vous pouvez laisser ce champ vide


Attention

Le nom de la page ne doit pas contenir d'espaces ni de caractères spéciaux car cela pourrait causer des problèmes par la suite au niveau de l'accès à la page


3.4.2. Supprimer une page


Afin de supprimer une page, vous devez cliquer sur le bouton "Supprimer cette page".

Un message vous avertira que la suppression de cette page entrainera également la suppression des différents liens que vous auriez pu ajouter. Si vous souhaitez toujours supprimer celui-ci, cliquez sur "OK".



3.4.3. Ajouter un lien


Pour ajouter un lien à votre page, cliquez sur le bouton "Ajouter un dashboard". Un pop-up apparaîtra.

Ce pop-up vous demandera l'information suivante :

  • Lien du Dashboard : Permets d'ajouter un lien que l'on souhaite afficher sur la page.


Info

Le lien du dashboard doit obligatoirement commencer par http:// ou https://


3.4.4. Supprimer un lien


Pour supprimer un lien, c'est très simple : cliquez sur la poubelle qui se trouve à droite du lien que vous désirez supprimer.


Un pop-up vous demandera une confirmation. Cliquez sur "OK" si vous n’avez pas changé d’avis.


3.4.5. Fonctionnalités

3.4.5.1. Cacher un dashboard

La fonctionnalité "Afficher" est utilisée pour masquer une page.

  • : Pour cacher un lien, cliquez sur l'oeil à droite du lien que vous souhaitez cacher. Celui-ci sera automatiquement retiré de l'ordre.
  • : Pour revoir le lien que vous avez caché, vous devez cliquer sur l'oeil barré. Celui-ci prendra la dernière position disponible.
3.4.5.2. Changer l'ordre des dashboards

Si vous souhaitez changer l'ordre des différents liens, cliquez sur les flèches situées à gauche de l’écran. La flèche vers le haut sert à monter le lien d’un cran, et la flèche vers le bas le fait descendre d’un cran.

5. Dashboard

5.1. Pages Dashing

5.1.1. Ajouter une page

Pour la création d'un nouveau dashboard sur dashing.io, cliquez sur la case "Ajouter". Une fenêtre pop-up apparaîtra.


Ce pop-up vous demandera l'information suivante :

  • Nom du dashboard : Définit le nom du fichier sur dashing.io qui sera également repris dans l'URL pour y accéder.

Attention

Le nom du dashboard ne doit pas contenir d'espaces ni de caractères spéciaux car cela pourrait causer des problèmes d’accès à la page.


5.1.2. Supprimer une page


Pour supprimer une page, cliquez sur le bouton "Supprimer ce dashboard".

Un message vous avertira que la suppression de ce dashboard entrainera également la suppression des différents Widgets que vous auriez pu ajouter. Si vous n’avez pas changé d’avis, cliquez sur "OK".



5.1.3. Ajouter un Widget



Pour l'ajout d'un nouveau Widget sur votre page dashing.io cliquez sur la case "Ajouter". Un pop-up apparaîtra.


Plusieurs informations vous seront demandées :

  • Nom de la frame : Nom que le Widget portera sur Octopus.
  • Type de widget : Type de widget que vous souhaitez utiliser.
  • Choix du Job : Sélection du job.
  • Un titre : Titre qui sera affiché sur le widget. (Facultatif)
  • Un sous-titre : Sous-titre qui sera affiché sur le widget. (Facultatif)
  • La largeur : Largeur en case du widget sur la page de 1 à 4.
  • La hauteur : Hauteur en case du widget sur la page de 1 à 4.
  • La couleur : Couleur de fond du widget.
  • Une redirection : Lien utilisé si l'on clique sur le widget. (Facultatif)

Si vous utilisez un Widget qui n'a pas besoin de Job, laissez le champ sur "Default".


Attention

Le Widget sélectionné doit être en adéquation avec le job que vous souhaitez utiliser.


Attention

Après l'ajout du widget, cliquez sur le bouton "construire" (5.1.6) pour appliquer les changements.


5.1.4. Supprimer un Widget


Pour la suppression d'un widget sur une page, positionnez votre curseur au-dessus du widget et ensuite cliquez sur la poubelle.

Attention

Après l'ajout du widget, cliquez sur le bouton "construire" (5.1.6) pour appliquer les changements.


5.1.5. Modifier un Widget


Future Update

La fonctionnalité de modification d'un widget n'est pas encore disponible. Toutefois, vous pouvez toujours en supprimer un (5.1.4), puis en recréer un nouveau (5.1.3).


5.1.6. Construire le dashboard



Le bouton « construire » est primordial, car l’ajout des widgets ne se réalisera que si l’on clique dessus.

Plusieurs couleurs peuvent apparaître sur le bouton "construire" :

  • Vert : Aucun problème, il n'y a donc pas besoin d'appuyer sur celui-ci.
  • Orange : Attention, une modification a été effectuée. Cliquez sur ce bouton pour appliquer ces changements.
  • Rouge : Une erreur a été rencontrée.

7. RemoteConfig


L'onglet "RemoteConfig" permet de faire l'installation des différents paquets qui seront nécessaires afin d'afficher différentes pages contrôlées par l'application Octopus

Prérequis

  • Un Raspberry PI

Fournissez les informations de votre Rasbperry PI

  • IP : Adresse IP utilisée pour rentrer en contact avec la raspberry PI
  • User : Nom d'utilisateur utilisé
  • Password : Mot de passe de l'utilisateur choisi

Ensuite pour chaque écran que dispose votre raspberry PI entrer les informations suivantes :

  • Type :
    • TabRotator : Permet d'utiliser l'extension TabRotator afin de pouvoir ajouter plusieurs onglets et les faire apparaître tous les X temps
    • Octopage : Permet d'utiliser une URL OctoPage*
  • Url* : Url de la page que vous souhaitez afficher depuis Octopage


8. Log

La majorité des logs générés par Octopus se retrouve sur cette page. Elles sont organisées en 3 Parties :

  • OctoLogin : Liste tous les logs en provenance de l'extension OctoLogin
  • OctoSystem : Liste tous les logs en provenance de dashing.io ou des requêtes comme la demande d'un nouveau token
  • Octopus : Liste tous les logs en provenance de l'application Web comme chaque manipulation

Plusieurs états pour les logs existent Success, Warning, Error, Info.


Sur la page d'accueil, vous pouvez apercevoir 3 graphiques afin d'avoir une vue globale des différents états des pages de log.


Afin de pouvoir afficher ces différents logs vous pouvez cliquer sur l'un des trois boutons se trouvant au-dessus des graphiques.


Tous les logs sont organisés de la même manière : [Date Heure][Utilisateur][Etat:Source] Etat : Message


Filtre : les différents boutons ci-contre vous permettront de filtrer le fichier log selon son état
Le bouton vider log permet de supprimer l'entièreté du fichier.
Cette action est irrévocable
